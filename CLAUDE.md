## 架构决策

### 多文件而非 SPA
参照 StormRelay 项目。每页独立 HTML + 共享 CSS/JS，零构建依赖。
录屏演示时 URL 变化直觉清晰；文件可独立调试；部署即复制。

### 状态管理
`localStorage` 以 `cc_` 前缀存储所有状态。核心函数：
- `getState(key, fallback)` / `setState(key, value)` — 读写状态
- `isScanned(pid)` / `markScanned(pid)` — 扫描状态查询/标记
- `getAdopted(pid)` / `adoptRisk(kw, pid)` / `adoptAllRisks(pid)` — 修复采用追踪
- `isAdopted(kw, pid)` — 单项采用状态查询
- `pulseNav()` — 采用后触发侧边栏"风险总览/历史"tab脉冲动画
- `trendChart(td, color)` — 返回内嵌趋势图表HTML（非modal）
- 售后状态：`aftersale_view_` / `aftersale_comp_` / `aftersale_text_` / `aftersale_history_` — 售后页面完整状态持久化
- `resetAll()` — 清空所有 `cc_*` 键值，重定向到概览

状态决定 UI 行为：未扫描商品在概览显示"待审核"灰色，售前直接进输入页；
已扫描商品显示风险数据，售前直接进结果页。采用的风险项在结果页、风险历史、风险总览中均有反映。

**重置触发**：双击侧边栏底部 "ClaimCheck v0.1 Demo" 文字。视觉上完全不像按钮。

### 商品排序
`PRODUCT_ORDER = ['jacket', 'base', 'tshirt']` 显式控制展示顺序。
冲锋衣和打底衫为主演示商品，短袖为辅助。

### 三维度审查
每个风险项标注 `dim` 字段，归属三大审查维度：
- **合规** — 违禁词、绝对化用语、医疗暗示（法律红线）
- **打假** — 虚假宣传争议、证据支撑不足、技术悖论、数据选择性披露（职业打假攻击面）
- **认知** — 认知鸿沟（消费者理解偏差导致客诉）

扫描动画三步：合规·违禁词与法规 → 防控·职业打假风险 → 认知·消费者理解偏差

### 扫描动画
总时长约 2.6 秒（三步进度条），配文"约 3 秒"。

### "采用"功能闭环
1. 售前扫描结果页：每个风险卡可单独"采用"或"全部采用"
2. 采用后 localStorage 持久化 `cc_adopted_{productId}` = `[kw1, kw2, ...]`
3. **采用动效**：点击后侧边栏"风险总览"和"风险历史"tab 绿色脉冲闪烁2次（`.sb-pulse` CSS动画）
4. 风险总览(archive)：SKU概况可折叠 + 全局修复追踪三列可折叠+按商品筛选 + AI学习折叠卡片
5. 风险历史(history)：内嵌完整趋势图表（不再需要点击打开modal）+ 风险列表已修复/删除线
6. 意义：系统越用越好用的迭代记录，部门总结用

### 售后状态持久化
售后话术页全部状态保存到 localStorage：输入文本、当前选择的客诉索引、视图状态（input/result）、追问对话历史。
导航离开再返回保持上次结果。"客户追问"输入框支持连续对话生成。

### 售前补充功能
扫描结果页增加"补充文案/图片"折叠区，支持追加文案并重新扫描。
原文高亮逻辑：优先匹配 `kw`，fallback 到 `original` 字段，自动跳过已高亮的重复关键词。

### 数据层
`PRODUCTS` 对象集中管理 3 个 SKU 的全部数据。支持动态创建新商品（保存到 localStorage）。
`DETAIL_IMAGES` 对象管理各商品的详情页图片路径（本地 `pic/` 文件夹）。
`TREND_DATA` 对象管理各商品的客诉趋势数据，支持 SVG 折线图。

### 图片策略
- 冲锋衣和打底衫使用 `pic/` 本地图片（演示可靠，无网络依赖）
- T恤和新创建商品使用 Unsplash fallback
- 上传功能术语为"AI 识图"（非OCR），支持多张图片预览

## 设计原则

1. **文案克制** — 用结构和图表说话，不堆文字
2. **AI润物细无声** — AI洞察、学习记录、自动关联，自然呈现而非功能入口
3. **演示优先** — 所有交互路径完整可点，状态可预设
4. **温暖专业** — Claude式暖色调，传递可信赖感
5. **状态一致** — 页面间不矛盾，扫没扫一目了然
6. **闭环体验** — 采用→修复进度→系统迭代，完整反馈循环
7. **时间合理** — mock数据不含"未来"信息，趋势用相对时间（峰值/近期），扫描历史用相对日期
8. **操作反馈** — 采用触发侧边栏脉冲、售后追问即时响应、补充文案重新扫描
